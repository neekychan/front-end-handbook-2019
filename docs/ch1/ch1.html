第1章 前端开发者是什么？

第一章 前端开发者是什么？

本章节阐述了前端开发的基本解释和前端开发者的规则。

前端web开发（也称作客户端开发）是网页和网页应用的生产HTML、CSS和JavaScript实践，以使用户能看见并与之直接交互。与前端开发关联的挑战是用于创建前端web的工具和技术不停地改变，并且开发者需要持续了解该领域是如何发展的。

设计一个网站的目的，就是确保当用户打开网站时，看到的信息被格式化成易于阅读且相关联的。实际上会更加复杂，现在用户使用大量不同的设备、不同的屏幕尺寸和分辨率，以至使设计师在设计网页的时候不得不考虑这些问题，他们必须确保页面能够在不同浏览器（夸浏览器）、不同的操作系统和不同的设备中正确展示，这就需要开发者有细心的规划。

- 维基百科

前端开发者...

前端开发者使用web技术（HTML、CSS、JavaScript）来设计和开发网站和web应用，这些技术通常运行在Open Web Platform或作为非网页平台环境的编译入口。(如React Native)

通过学习构建依赖于HTML, CSS, and JavaScript的网站或网页应用来进入前端开发领域。通常运行在web浏览器中，但也能运行于无头浏览器、WebView或者原生运行环境中。下面将介绍四种运行时方案：

Web浏览器（最常见的）

Web浏览器是一款用于检索、展示、遍历WWW信息的软件。通常，浏览器运行在桌面以及手提电脑、平板电脑、手机，但最近浏览器几乎能运行在任何设备上。（冰箱、汽车等等）

最常见的Web浏览器（按照最常用的顺序排列）：

  • Chrome
  • Safari
  • Internet Explorer (备注: 不含Edge, 特指 IE 9 至 IE 11)
  • Firefox
  • Edge

无头浏览器

Headless浏览器指不包含图形界面的浏览器，通过命令行界面以编码的方式进行控制以达到网页自动化的目的（功能测试、网页爬取、单元测试）。

最常见的无头浏览器：

  • Headless Chromium
  • Zombie
  • slimerjs
  • puppeteer

Webviews

Webviews用在原生操作系统上，在原生应用中运行网页。你可以将webviews想象成是网页浏览器的一个iframe或一个单标签，它被嵌入在设备里所运行的原生应用中。(如iOS、android、windows)。

最常见的webview开发解决方案：

  • Cordova (手机/平板的原生应用)
  • NW.js (桌面应用)
  • Electron (桌面应用)

Native from Web Tech

最后，前端开发者可以从网页浏览器开发中所学的知识，不受浏览器引擎的限制来编写代码。最近，开发环境正被设想成可以利用网页技术（CSS、JavaScript）且不使用网页引擎来构建原生应用。

这些环境部分的列子：

  • Flutter
  • React Native
  • NativeScript

备注：

1、确保你清晰理解“web platform”的含义。阅读“Open Web Platform”的维基百科页面。探索web platform相关的众多技术。
